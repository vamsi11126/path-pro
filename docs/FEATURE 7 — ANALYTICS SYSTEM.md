# **⭐ 1\. What Feature 7 Is**

Feature 7 is the **Analytics & Insights Engine** of Learnify.  
 It processes all data generated by:

* learning sessions,

* review sessions,

* topic progress changes,

* time spent studying,

* ratings (SM‑2 quality scores),

* unlocking patterns.

Then converts this raw data into:

* charts,

* insights,

* weaknesses detection,

* weekly learning summaries,

* subject-level analytics,

* overall progress measurement.

This feature is the **"brain monitor"** of Learnify.

---

# **🎯 2\. Primary Use Case of Feature 7**

**To give students deep visibility into their learning performance**, answering questions like:

* *How much did I study this week?*

* *Which topics am I weak at?*

* *Which subjects need more attention?*

* *What are my upcoming reviews?*

* *What percentage of each subject is mastered?*

Without analytics, Learnify would feel like a static to-do list.  
 With analytics, it becomes a **personalized learning intelligence system**.

# **⚙️ 4\. How Feature 7 Works Internally (Processing Flow)**

Feature 7 gathers data from two places:

1. `topics` table → statuses, upcoming reviews

2. `study_logs` table → actual learning activity

Then produces multiple layers of analytics.

## **🔸 Step 1 — Study Time Analytics**

From `study_logs.duration_seconds`, grouped by:

* day,

* week,

* subject.

Charts produced:

* Weekly Study Time

* Daily Streak Chart

---

## **🔸 Step 2 — Performance Analytics (Weak Topic Detection)**

From `study_logs.performance_rating`:

* compute average score per topic.

* compute mastery trend.

Weak topics \= topics with:

* avg rating \< 3.0

* high repetition but low mastery

Displayed in Subject Dashboard:

* “Weakest Topics” section

---

## **🔸 Step 3 — Progress Analytics (Status Breakdown)**

From `topics.status` for each subject:

* count mastered

* count reviewing

* count learning

* count available

* count locked

Compute:  
 **subject\_progress \= mastered / total\_topics**

Displayed as:

* progress bars,

* donut charts,

* graph node color coding.

---

## **🔸 Step 4 — Review Forecasting**

Using `topics.next_review_at`:

* upcoming reviews this week

* overdue reviews count

* heatmap of scheduled reviews

Displayed in:

* dashboards

* review queue

---

## **🔸 Step 5 — Knowledge Graph Difficulty Analytics**

Using:

* number of edges per topic,

* graph depth,

* unlock patterns.

Analytics can estimate:

* critical path topics,

* dependency bottlenecks,

* hardest topics.

Used mostly in advanced dashboards.

---

# **🧩 5\. What Data Feature 7 Uses (`study_logs`)**

Each review or learning session inserts:

* user\_id

* topic\_id

* duration\_seconds

* performance\_rating (0–5)

* reviewed\_at

This table is the **core engine** behind analytics.

# **📐 6\. What Exactly You Must Implement (Developer Breakdown)**

Feature 7 requires both **backend queries** and **frontend charts**.

## **✔ Backend Functions Needed**

### **1\. `getStudyTimeByWeek(userId)`**

Group study\_logs by week.

### **2\. `getWeakTopics(subjectId)`**

Compute avg rating per topic.

### **3\. `getSubjectProgress(subjectId)`**

Count mastered / total topics.

### **4\. `getUpcomingReviews(subjectId)`**

Query topics with near `next_review_at`.

### **5\. `getRetentionHealth(subjectId)`**

Based on SM‑2 performance.

### **6\. `getHeatmapData(userId)`**

Daily learning activity.
