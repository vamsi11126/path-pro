# Contribution Guidelines

Welcome to the Learnify project! We appreciate your interest in contributing to our project. Please follow the guidelines below to ensure a smooth and collaborative workflow.

## Team Workflow
1. **Communication**: We use [Slack](#) for real-time communication. Join the channel for updates.
2. **Issue Tracking**: Use GitHub issues to track bugs, feature requests, and tasks. Make sure to tag your issue appropriately.

## Branching Strategy
1. **Main Branch**: The `main` branch contains the stable version of the project.
2. **Feature Branches**: When working on a new feature, create a branch from `main` with the naming convention `feature/<feature-name>`.
3. **Bug Fix Branches**: For bug fixes, create branches with the naming convention `bugfix/<bug-name>`.
4. **Release Branches**: For preparing releases, create branches with the naming convention `release/<version-number>`.

## Pull Request (PR) Process
1. **Create a Pull Request**: Once your feature or bug fix is complete, create a pull request against the `main` branch.
2. **Describe Your Changes**: Provide a clear description of the changes made, referencing the issue number if applicable.
3. **Request Reviews**: Assign reviewers from the team to get feedback on your PR.
4. **Merge When Approved**: Once approved, you can merge your PR into the `main` branch.

## Development Setup Instructions
1. **Clone the Repository**: Execute the command `git clone https://github.com/SanthoshReddy352/Learnify.git`.
2. **Install Dependencies**: Navigate to the project directory and run `npm install` to install all required dependencies.
3. **Start the Development Server**: Use `npm start` to start the development server.
4. **Run Tests**: To run tests, use the command `npm test`.

Thank you for your contributions! Together, let's make Learnify a great project.
